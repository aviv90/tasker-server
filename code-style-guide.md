# Code Style Guide

1. פרומפטים - נקיים, מסודרים, ללא חזרתיות וללא כללים מבלבלים. חשוב שהם גם יהיו מופרדים מהקוד.
2. שמירה על כל הסטנדרטים הגבוהים ביותר של קידוד: כל ה-best practices, סקיילביליות, SSOT, מניעת כפילויות, קוד נקי, אופטימלי, ביצועים הכי גבוהים שאפשר, טסטביליות, Centralized logic, הימנעות מ-Hardcoded values, מודולריות וכל השאר, כל העקרונות המיטביים.
3. ארכיטקטורה אידאלית וברורה - הפרדת תחומי אחריות בצורה המיטבית.
4. שמירה על כל ה-Cursor rules שלנו וה-rules שלך, ווידוא שאנחנו עומדים בכולם.
5. ביצועים מקסימליים.
6. Acks נכונים, מדויקים ועקביים בכל המקרים.
7. הימנעות ממעבר ספונטני לכלים אחרים מאלה שהתבקשו.
8. ספקי ברירת מחדל כנדרש, ושימוש בספקים אחרים רק בבקשה מפורשת של המשתמש. אפשרות לנסיון חוזר עם ספק אחר (אם התבקש במפורש), או עם אותו ספק, כולל עם בקשות לשינויים. ללא fallbacks אוטומטיים!
9. גישה סלחנית - משתמשים יכולים לכתוב בשפות שונות, בשפה טבעית לחלוטין ועם שגיאות. המערכת צריכה להגיב גם לבקשת כאלה ולהבין באופן מדויק את כוונת המשתמש.
10. שליחת כל שגיאה שקורית במהלך התהליך ללקוח כפי שהיא, as-is.
